Installing PowershellGet

- Open Windows Powershell in Admin mode,

- Write the following command and hit Enter, it will tell you what is the current version you have 
 Get-Module -Name PowerShellGet -ListAvailable | Select-Object -Property Name, Version, Path

Name          Version Path
----          ------- ----
PowerShellGet 1.0.0.1 C:\Program Files\WindowsPowerShell\Modules\PowerShellGet\1.0.0.1\PowerShellGet.psd1

-Upgrade the PowerShellGet to latest version, write the following command and Hit Enter
Install-Module PowerShellGet -Force 

-To Check the latest version, Write the following command and hit Enter
Get-Module -Name PowerShellGet -ListAvailable | Select-Object -Property Name, Version, Path

Name          Version Path
----          ------- ----
PowerShellGet 2.2.5   C:\Program Files\WindowsPowerShell\Modules\PowerShellGet\2.2.5\PowerShellGet.psd1
PowerShellGet 1.0.0.1 C:\Program Files\WindowsPowerShell\Modules\PowerShellGet\1.0.0.1\PowerShellGet.psd1

-Before using Azure Powershell functionality, Since the default policy on for script execution on windows powershell is RESTRICTED 
-for running scripts, we need to change it , type the following script and hit enter >

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned

--It will show following, enter A (Yes to All) and hit enter

Execution Policy Change

execution policy might expose you to the security risks described in the
about_Execution_Policies help topic at https:/go.microsoft.com/fwlink/?LinkID=135170. Do you
want to change the execution policy?
[Y] Yes  [A] Yes to All  [N] No  [L] No to All  [S] Suspend  [?] Help (default is "N"):


-- Download Azure RM (Resource Manager)module now, type the following script and hit enter
**AllowClobber Switch makes sure, we will get full and complete Installtion of Azure RM module from Galary

Install-Module -Name AzureRM -AllowClobber

--You will get following Message, type Y and Hit Enter

Untrusted repository
You are installing the modules from an untrusted repository. If you trust this repository,
change its InstallationPolicy value by running the Set-PSRepository cmdlet. Are you sure you
want to install the modules from 'PSGallery'?
[Y] Yes  [A] Yes to All  [N] No  [L] No to All  [S] Suspend  [?] Help (default is "N"):

--If you already have any version of Azure RM installed, it will give following warning 

WARNING: Version '5.7.0' of module 'AzureRM' is already installed at 'C:\Program
Files\WindowsPowerShell\Modules\AzureRM\5.7.0'. To install version '6.13.2', run
Install-Module and add the -Force parameter, this command will install version '6.13.2'
side-by-side with version '5.7.0'.

--Type the following command and hit enter
Install-Module -Name AzureRM -Force

--After Installation type the following command and Hit Enter

 Get-Module -Name AzureRM -ListAvailable | Select-Object -Property Name, Version, Path

Name    Version Path
----    ------- ----
AzureRM 6.13.2  C:\Program Files\WindowsPowerShell\Modules\AzureRM\6.13.2\AzureRM.psd1
AzureRM 5.7.0   C:\Program Files\WindowsPowerShell\Modules\AzureRM\5.7.0\AzureRM.psd1

--Loading AzureRM Module
-- Before Loading Azure Powershell Module, it must be first Loaded into Powershell Session
-- Import Module is used to Load Powershell Modules so that their commands can be accessed
-- Pulls the module into active session, so the commandLets can be accessed
-- Type the following command and Hit Enter

Import-Module -Name AzureRM
